**Note:** The hero uses `logo` (not `title_line1`/`title_line2`) — displayed as large centered text. Reviews use the scrolling shape (no `rating` field, has `avatar`).

## Tailwind v4 Gotchas

- `bg-linear-to-br` not `bg-gradient-to-br`; `mix-blend-screen` not `mix-blend-mode-screen`
- Numeric scale only: `min-h-55`, `md:w-88`, `z-60`, `max-w-350`
- Arbitrary calc: `w-[calc((100%-2rem)/3)]` — no underscores
- No dynamic Tailwind classes (e.g. `` `grid-cols-${n}` `` breaks JIT) — use const maps with static strings

## Known Gotchas

- `params` must be `Promise<{ locale: string }>`, cast with `as { locale: Locale }`
- Framer Motion ease in variants → type errors; use `ease: [0.16, 1, 0.3, 1] as const`
- `React.FormEvent` deprecated in React 19 — use `{ preventDefault(): void }`
- `ScrollProgress z-60` > `Navbar z-50`
- `app/page.tsx` must export a default or build fails
- Navbar is a client component — do not revert to server
- Menu uses `AnimatePresence mode="sync"` — do not change to `"wait"` (causes SSR hydration mismatch)
- Contact.tsx dispatches `window.dispatchEvent(new Event("open-reservation"))` — used by the Reservation modal section if installed

## Personalization Workflow

**Paste this file into a new Claude Code conversation** opened in this project directory.

### Step 1 — Restaurant Details

Ask the user ALL of the following **in a single numbered list** (don't start any work yet):

1. Restaurant name
2. Cuisine type (e.g. "Italian", "Portuguese", "Steakhouse") — for JSON-LD and context
3. Tagline / one-sentence description
4. City and country
5. Phone number (with country code)
6. WhatsApp number (digits only) — if applicable
7. Email address
8. Physical address
9. Opening hours (e.g. "Tue–Sun 12pm–11pm")
10. Google Maps embed URL (from Google Maps → Share → Embed a map → copy the `src` URL)
11. Google Maps directions URL (from Google Maps → Directions → copy the URL)
12. Site URL (Vercel deployment or custom domain)
13. 3 hero stats — value + label each (e.g. "5K+ / Meals Served, 4.8★ / Average Rating, 12+ / Years Open")

### Step 2 — Section Content

After Step 1 answers, ask for all section content in one message:

**About:** 2 short paragraphs (restaurant story, philosophy, what makes it special), 3 stats (value + label)

**Menu:** For each category (starters, mains, desserts, drinks, etc.):
- Category name
- 3–5 items per category: name, short description, price
- Optional: image URL per item (leave empty if not available)

**Reviews:** 6–9 testimonials — quote, guest name, source (e.g. "Via Google", "Via TripAdvisor")

**FAQ:** 6 questions + answers (reservations, hours, dietary options, parking, events, delivery)

**Contact:** Confirm phone, email, address, hours from Step 1. Contact body text (1 sentence inviting guests).

**Footer:** Confirm restaurant name, tagline, address, hours, phone, email, copyright year

**FloatingCTA labels (if installed):** Call label, WhatsApp label, Book label

**ReserveBar labels (if installed):** Reserve CTA label, Order online CTA label, order URL

### Step 3 — Apply All Content

1. **Dictionaries** — Fill `dictionaries/en.json` with all content. If `dictionaries/pt.json` exists, fill it too (ask user for translations or translate and confirm).

2. **Layout & Metadata** — In `app/[locale]/layout.tsx`:
   - `SITE_URL` → actual domain
   - `title` → "Restaurant Name — Short descriptor"
   - `description` → one-sentence description (both languages if i18n)
   - `jsonLd` → name, description, email, telephone, address, servesCuisine

3. **SEO** — `app/robots.ts` and `app/sitemap.ts`: replace `YOUR_DOMAIN`

4. **Clean up** — Remove all `// TODO: TEMPLATE` comments

### Step 4 — Images

| File | Description |
