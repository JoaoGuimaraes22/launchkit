## Tailwind v4 Gotchas

- `bg-linear-to-br` not `bg-gradient-to-br`; `mix-blend-screen` not `mix-blend-mode-screen`
- Numeric scale only: `min-h-55`, `md:w-88`, `z-60`, `max-w-350`
- Arbitrary calc: `w-[calc((100%-2rem)/3)]` — no underscores
- No dynamic Tailwind classes (e.g. `` `grid-cols-${n}` `` breaks JIT) — use const maps with static strings

## Known Gotchas

- `params` must be `Promise<{ locale: string }>`, cast with `as { locale: Locale }`
- Framer Motion ease in variants → type errors; use `ease: [0.16, 1, 0.3, 1] as const`
- `React.FormEvent` deprecated in React 19 — use `{ preventDefault(): void }`
- `ScrollProgress z-60` > `Navbar z-50`
- `app/page.tsx` must export a default or build fails
- Navbar is a client component — do not revert to server

## Placeholder Markers

All `YOUR_*` strings must be replaced before deployment:

```
YOUR_BUSINESS          Business name (navbar, footer, metadata, JSON-LD)
YOUR_PHONE             Phone number with country code
YOUR_WHATSAPP_NUMBER   WhatsApp number (digits only, no spaces)
YOUR_EMAIL             Email address
YOUR_ADDRESS           Physical address
YOUR_DOMAIN            Site URL in layout.tsx metadata
```

Sections may add more: `YOUR_CALENDLY_URL`, `YOUR_GOOGLE_REVIEW_URL`, `YOUR_FORMSPREE_ID`, etc.

## Personalization Workflow

**Paste this file into a new Claude Code conversation** opened in this project directory.

### Step 1 — Business Details

Ask the user ALL of the following **in a single numbered list** (don't start any work yet):

1. Business name
2. Business type (e.g. "dental clinic", "hair salon") — for context
3. Tagline / one-sentence description
4. City and country
5. Phone number (with country code)
6. WhatsApp number (digits only)
7. Email address
8. Physical address
9. Business hours
10. Google Maps link for the address
11. Site URL (Vercel deployment or custom domain)
12. 3 hero stats — value + label each (e.g. "+15 / Years Experience")

### Step 2 — Section Content

After Step 1 answers, ask for all section content in one message:

**Hero:** Two headline lines (uppercase), tagline, primary CTA label, secondary CTA label

**About:** 2 short paragraphs, 3 stats (value + label)

**Services:** 6 services — emoji icon, title, 1-sentence description each

**Reviews:** 4–6 testimonials — quote, name, role, star rating (1–5)

**FAQ:** 6 questions + answers

**Contact:** Confirm contact info from Step 1, contact body text (1 sentence)

**Footer:** Confirm business name, tagline, address, hours, phone, email, copyright year

**FloatingCTA labels (if installed):** Call label, WhatsApp label, Book label

### Step 3 — Apply All Content

1. **Dictionaries** — Fill `dictionaries/en.json` with all content. If `dictionaries/pt.json` exists, fill it too (ask user for translations or translate and confirm).

2. **Layout & Metadata** — In `app/[locale]/layout.tsx`:
   - `SITE_URL` → actual domain
   - `title` → "Business Name — Short descriptor"
   - `description` → one-sentence description (both languages if i18n)
   - `jsonLd` → name, email, telephone, address

3. **Contact API route** (if `app/api/contact/route.ts` exists): set `TO_EMAIL`

4. **SEO** — `app/robots.ts` and `app/sitemap.ts`: replace `YOUR_DOMAIN`

5. **Clean up** — Remove all `// TODO: TEMPLATE` comments
